## 1. High-Level Overview

### What is this project?

**DSPDM** is a **metadata-driven, multi-tenant data management platform** for the **oil & gas industry**. It manages entities like Wells, Wellbores, Fields, Areas, Platforms, Pipelines, Equipment Specifications, Production Volumes, and dozens more — all **without hardcoded entity classes**.

### What problem does it solve?

Traditional systems require a Java class per database table (like JPA/Hibernate entities). DSPDM takes a radically different approach: **the entire data model is driven by metadata stored in the database itself**. When a customer needs a new business object (e.g., a new table for "Pipeline Inspection"), they don't need developer intervention — the system reads its own `BUSINESS_OBJECT` and `BUSINESS_OBJECT_ATTR` tables and dynamically generates SQL, builds DTOs, and exposes REST APIs for the new table **at runtime**.

## 2. Architecture & Flow

### Architecture Style: **Layered Monolith with Dynamic Schema**

Not a microservice — it's a **monolithic multi-module Maven project** deployed as a single WAR on WildFly/Teiid, but with clean layered separation:

| Layer | Module | Responsibility |
|-------|--------|----------------|
| **REST API** | `dspdm.msp.mainservice` | 75+ JAX-RS endpoints, auth, validation, WAF bypass |
| **Service Facade** | `dspdm-service` | Zero-logic pass-through (clean interface for REST layer) |
| **Business Logic** | `dspdm-delegate` | Transaction management (JTA/Bitronix), orchestration, recursive parent-child ops, change tracking |
| **Data Access** | `dspdm-repo` | Metadata-driven dynamic SQL generation, JDBC, batch processing |
| **Virtual DB** | `dspdm-vdb` | Teiid VDB with 100+ virtual views and 50+ UDFs |
| **Cross-cutting** | `dspdm-common`, `dspdm-redis`, `dspdm-unit-conversion`, `dspdm-import-export` | Shared DTOs, caching, unit math, Excel I/O |

### Complete Request Flow (Read)

```
1. HTTP POST /api/v1/common   (JSON BOQuery body)
   │
2. TenantFilter          → Extract tenant from JWT, validate
3. RequestBodyVerifyFilter→ Validate JSON, handle base64 encoding
   │
4. MainserviceImpl.getCustomResponsePost()
   │  ├─ getCurrentUserExecutionContext()   → JWT validation + API permission check
   │  ├─ EquipmentHelper check             → detect equipment BOs, set flag
   │  ├─ BusinessObjectBuilder.build()     → BOQuery → BusinessObjectInfo
   │  │    ├─ WAFRulesByPasser.byPassWafRules()  → decode WAF tokens
   │  │    ├─ Validate & build pagination, orderBy, selectList
   │  │    ├─ Validate & build criteriaFilters, filterGroups
   │  │    └─ Validate & build joins, unit policy
   │  ├─ DTOHelper.convertDataTypeForFilters()  → cast filter values
   │  │
5. │  └─ dynamicReadService.readWithDetails()
   │       │
6. │       └─ DynamicReadService (facade) → delegates via BusinessDelegateFactory
   │           │
7. │           └─ BusinessObjectReadDelegate (outer proxy)
   │               ├─ Exception handling
   │               └─ BusinessObjectReadDelegateImpl (real logic)
   │                   ├─ Reads count
   │                   ├─ Reads main BO data
   │                   ├─ Reads metadata, constraints
   │                   ├─ Reads reference data (parent lookups)
   │                   └─ Fills parent/child hierarchies
   │                       │
8. │                       └─ DynamicDAOFactory.getDynamicDAO("WELL")
   │                           │
9. │                           └─ DynamicDAO.read(businessObjectInfo, connection)
   │                               ├─ Build SQL dynamically from metadata
   │                               ├─ PreparedStatement with fetch-size optimization
   │                               ├─ Bind parameters
   │                               ├─ executeQuery()
   │                               └─ Extract ResultSet → List<DynamicDTO>
   │
10.│  UnitHelper.unitConvertAfterRead()  → feet→meters, psi→bar, etc.
   │
11.└─ new DSPDMResponse(SUCCESS, data).getResponse()  → JSON response
```

**Example SQL generated dynamically at step 9:**
```sql
SELECT WELL_ID, UWI, WELL_NAME, FIELD_ID, R_WELL_STATUS_ID
FROM   WELL
WHERE  FIELD_ID = ? AND R_WELL_STATUS_ID IN (?, ?, ?)
ORDER BY WELL_NAME ASC
OFFSET (?) ROWS FETCH NEXT (?) ROWS ONLY
```

### Complete Request Flow (Write/Save)

```
1. HTTP POST /api/v1/save  { "WELL": { "data": [...] } }
   │
2. Filters (Tenant + Body validation)
   │
3. MainserviceImpl.saveOrUpdate()
   │  ├─ DTOHelper.buildBONameAndBOListMapFromRequestJSON()
   │  │    → Parse JSON → Map<String, List<DynamicDTO>>
   │  │    → Handle nested children recursively
   │  │    → Apply WAF rules
   │  │
4. │  └─ dynamicWriteService.saveOrUpdate(map, ctx)
   │       │
5. │       └─ BusinessObjectWriteDelegate (outer proxy)
   │           ├─ beginTransaction()    ← JTA/Bitronix
   │           ├─ BusinessObjectWriteDelegateImpl
   │           │    ├─ Validate entity types
   │           │    ├─ Split into insert list (no PK) + update list (has PK)
   │           │    ├─ For inserts: sequence gen → batch INSERT
   │           │    ├─ For updates: read-before-update → batch UPDATE
   │           │    ├─ Handle children recursively
   │           │    ├─ Change history tracking
   │           │    └─ Search index creation
   │           ├─ commitTransaction()   ← on success
   │           └─ rollbackTransaction() ← on failure
   │
6. └─ DSPDMResponse(SUCCESS, saveResult)
```

## 4. Core Java Concepts Used

### Collections — Where and Why (Actual Project Usage)

| Collection | Where Used | Why This Specific Type | Real Impact |
|------------|-----------|----------------------|--------------|
| **`TreeMap<String, Object>`** | `DynamicDTO` extends this | Case-insensitive key lookup for column names | Eliminates bugs like `dto.get("WELL_ID")` vs `dto.get("well_id")` — both work |
| **`LinkedHashMap<String, ?>`** | `SaveResultDTO.insertCounts`, `ExecutionContext.sqlStatementsMap` | Preserves insertion order | UI shows BOs processed in save order, SQL debug shows execution sequence |
| **`LinkedHashSet<String>`** | `DynamicDTO.orderedKeySet` | Column order preservation + uniqueness | Excel exports maintain column order from first query |
| **`TreeSet<Integer>`** | `Pagination.pages` | Sorted page numbers | Deterministic pagination: `[1,3,2]` becomes `[1,2,3]` automatically |
| **`ConcurrentHashMap`** | `DynamicDAOFactory.daoMap`, `MSMHelper.unitCache` | Thread-safe shared singletons | Multiple request threads safely access DAO registry without locks |
| **`ArrayList`** | 90% of lists in codebase | General-purpose ordered collections | BO attribute lists, filter lists, result lists — fast indexed access |
| **`Collections.unmodifiableList()`** | All `DSPDMConstants` lists | Immutable shared constants | Prevents `DSPDMConstants.BoName.SERVICE_DB_BO_NAMES.add("HACK")` at runtime |
| **`Vector`** (legacy) | Some old code paths | Synchronized ArrayList | Being phased out — use `Collections.synchronizedList()` instead |

### equals() / hashCode() — Actual Project Usage

```java
// DynamicPK — Composite primary key with proper identity semantics
public class DynamicPK {
    private String type;        // BO name (e.g., "WELL")
    private Object[] pkValues;  // composite PK values (e.g., [42, "ABC-123"])

    @Override
    public boolean equals(Object o) {
        DynamicPK other = (DynamicPK) o;
        return Objects.equals(type, other.type) 
            && Arrays.equals(pkValues, other.pkValues);  // Deep array comparison
    }

    @Override
    public int hashCode() {
        return Objects.hash(type) + Arrays.hashCode(pkValues);  // Consistent hashing
    }
}
```

**Real-world usage in the project:**
- **Read-before-update**: `Map<DynamicPK, DynamicDTO> existingRecords` — lookup current DB state by PK
- **Duplicate detection**: `Set<DynamicPK> processedKeys` — prevent processing same record twice in batch
- **Change tracking**: Compare `oldDTO.equals(newDTO)` to determine if UPDATE is needed
- **Cache keys**: Redis uses `DynamicPK.toString()` for reference data cache keys

**AliasedColumn equals/hashCode:**
```java
// Used in SELECT lists and ORDER BY validation
public class AliasedColumn {
    @Override
    public boolean equals(Object o) {
        return Objects.equals(boAttrName, other.boAttrName) 
            && Objects.equals(alias, other.alias);
    }
}
```
Essential for `Set<AliasedColumn>` duplicate detection in complex SELECT queries.

### Immutability — Defensive Programming

**Metadata protection after startup:**
```java
// MetadataCacheDAO.makeAllMetadataListsUnmodifiable()
this.boAttrList = Collections.unmodifiableList(this.boAttrList);
this.activeBoAttrList = Collections.unmodifiableList(this.activeBoAttrList);
this.pkColumnsList = Collections.unmodifiableList(this.pkColumnsList);
// Prevents: daoFactory.getDAO("WELL").getBoAttrList().clear(); ← RuntimeException
```

**Constants protection:**
```java
// DSPDMConstants — all lists are immutable
public static final List<String> SERVICE_DB_BO_NAMES = 
    Collections.unmodifiableList(Arrays.asList(
        "BUSINESS_OBJECT", "BUSINESS_OBJECT_ATTR", "DSPDM_CONFIG_INFO"
    ));
// Prevents: SERVICE_DB_BO_NAMES.add("HACK"); ← UnsupportedOperationException
```

**Value objects:**
- **`DynamicPK`**: No setters after construction — hashCode/equals stable
- **`TenantContext`**: Immutable `tenantId` + `isMultiTenant` flag
- **`OrderBy`**: Immutable sort specification (boAttrName + direction)

**Why it matters:** Shared singletons (`DynamicDAOFactory`, metadata caches) are safely accessed by concurrent requests without defensive copying.

### `final` / `static` — Real Usage Patterns

| Usage | Example | Why | Performance Impact |
|-------|---------|-----|--------------------|
| `static` factory methods | `ExecutionContext.getSystemUserExecutionContext()` | Controlled object creation, validation | No constructor overhead |
| `static final` constants | 500+ in `DSPDMConstants.BoAttrName.WELL_ID` | Type safety, IDE autocomplete, refactoring safety | JVM inlines at compile time |
| `static` utility methods | `StringUtils.hasValue()`, `DTOHelper.applyWafRule()` | No instance state needed | No object allocation |
| `volatile` + `static` | `ConfigProperties.instance`, `DynamicDAOFactory.instance` | Thread-safe lazy singletons | Memory barrier ensures visibility |
| `final` fields | `DynamicDTO.type`, `DynamicPK.pkValues` | Immutable after construction | JIT optimization, no defensive copying |
| `static` initializers | `DSPDMConstants` nested interfaces | Eager initialization of constants | Class loading time cost |

### Exception Handling — Unchecked Strategy with Smart Wrapping

```java
// The entire project uses unchecked exceptions only
RuntimeException
  └── DSPDMException                // general — thrown everywhere
       ├── DSPDMBatchException      // wraps multiple SQL exceptions from batch ops
       └── DSPDMValidationException // validation-specific errors
```

**Smart SQL exception unwrapping (real project code):**
```java
// DSPDMException.extractSQLException() — handles Teiid wrapper exceptions
public static DSPDMException extractSQLException(Throwable throwable) {
    Throwable cause = throwable;
    List<SQLException> sqlExceptions = new ArrayList<>();
    
    // Traverse cause chain to find all SQL exceptions
    while (cause != null) {
        if (cause instanceof SQLException) {
            sqlExceptions.add((SQLException) cause);
        }
        cause = cause.getCause();
    }
    
    // Convert SQL state codes to user messages
    if (sqlExceptions.size() == 1) {
        return new DSPDMException(translateSQLState(sqlExceptions.get(0)));
    } else {
        return new DSPDMBatchException(sqlExceptions);  // Multiple failures
    }
}
```

**Exception translation examples:**
- `23505` (unique violation) → "Record already exists with this ID"
- `23503` (foreign key violation) → "Cannot delete - record is referenced by other data"
- `42P01` (undefined table) → "Business object 'WELL' does not exist"

**Why unchecked exceptions:**
- No `throws SQLException` on 200+ DAO methods
- Clean interfaces — `IDynamicReadService.read(BusinessObjectInfo)` not `...throws DSPDMException`
- Exception handling centralized at REST layer (`MainserviceImpl` catch blocks)
- Async/callback code doesn't need exception propagation

### Object Lifecycle & Memory Management

#### `ExecutionContext` Lifecycle (Per-Request Pattern)
```
1. CREATED:    getCurrentUserExecutionContext(headers, permission)  ← factory method
2. POPULATED:  JWT parsed → userName, tenantId, locale, timezone, permissions
3. PASSED:     Through every layer as explicit parameter (NOT ThreadLocal!)
4. ENRICHED:   SQL statements, timing, operation counts accumulated during request
5. LOGGED:     logTotalProcessingTime() — full request breakdown
6. DISCARDED:  Goes out of scope after HTTP response sent (GC eligible)
```

#### Memory-Conscious Design Patterns

**Large collections streaming:**
```java
// Excel export — only keep window in memory
SXSSFWorkbook workbook = new SXSSFWorkbook(1000);  // 1000-row window
for (DynamicDTO dto : largeResultSet) {
    // Older rows automatically flushed to disk
}
```

**Connection lifecycle (explicit resource management):**
```java
// DynamicDAO pattern — no connection pooling leaks
Connection connection = null;
try {
    connection = getReadOnlyConnection(boType, executionContext);
    return executeQuery(connection, sql, params);
} finally {
    closeConnection(boType, connection, executionContext);  // Always released
```

**Batch processing memory control:**
```java

    List<DynamicDTO> batch = dtos.subList(i, Math.min(i + batchSize, dtos.size()));

}
```

**Metadata caching strategy:**
- **Startup**: Load all metadata into memory (2-5 MB typically)
- **Runtime**: Zero DB hits for schema info
- **Refresh**: Clear + reload on `/refreshMetadata` (GC clears old objects)

### Why `DynamicDTO` extends `TreeMap` (not `HashMap`)

```java
public class DynamicDTO extends TreeMap<String, Object> implements IDTO<DynamicPK> {
    public DynamicDTO(String type) {
        super(String.CASE_INSENSITIVE_ORDER);  // ← the key insight
    }
}
```

Database column names come in mixed case (`Well_ID`, `WELL_ID`, `well_id`). Using `TreeMap` with `CASE_INSENSITIVE_ORDER` means:
- `dto.get("WELL_ID")` and `dto.get("well_id")` return the **same value**
- **Trade-off**: O(log n) lookups instead of O(1) HashMap
- **Benefit**: Eliminates an entire class of case-mismatch bugs

### Thread Safety Approach — Per-Component Analysis

**The system follows "thread-per-request" model with shared singletons:**

| Component | Thread Safe? | Implementation Details | Real Impact |
|-----------|-------------|----------------------|-------------|
| **`ExecutionContext`** | ❌ No | Mutable fields, no `volatile`, no `synchronized` | Each HTTP request thread gets its own instance — never shared |
| **`DynamicDTO`** | ❌ No | Extends `TreeMap` (not thread-safe) | Each request builds its own DTOs — no cross-thread sharing |
| **`DynamicDAO` instances** | ✅ Stateless | All state comes from method parameters | 50+ request threads safely call `read(BusinessObjectInfo)` on same DAO |
| **`ConfigProperties`** | ✅ Singleton | `volatile ConfigProperties instance` + double-checked locking | Thread-safe lazy initialization, properties are effectively final |
| **`DynamicDAOFactory`** | ✅ Singleton | `ConcurrentHashMap<String, DynamicDAO> daoMap` | Multiple threads safely get DAOs: `getDynamicDAO("WELL")` |
| **`DSPDMRedisCacheUtils`** | ✅ Singleton | `JedisCluster` is thread-safe, operations are atomic | Cache reads/writes from concurrent requests don't interfere |
| **`DSPDMConstants` lists** | ✅ Immutable | `Collections.unmodifiableList()` — no mutation possible | Safe to read from any thread without copying |
| **`MSMHelper` unit cache** | ✅ Immutable after init | All `HashMap`s populated at startup, then read-only | Unit conversions safe across threads |

### Thread-Safe Patterns Used Throughout Codebase

#### 1. Double-Checked Locking Singleton (used 8+ times)
```java
// ConfigProperties, DynamicDAOFactory, DSPDMRedisCacheUtils, etc.
private static volatile ConfigProperties instance = null;

public static ConfigProperties getInstance() {
    if (instance == null) {                          // 1st check: fast path (no lock)
        synchronized (ConfigProperties.class) {       // 2nd: acquire class lock
            if (instance == null) {                    // 3rd: double-check with lock
                instance = new ConfigProperties();     // 4th: safe construction
            }
        }
    }
    return instance;  // volatile read ensures full visibility
}
```

## 6. Design Decisions (VERY IMPORTANT)

### Decision 1: Dynamic DTO Instead of Static Entities

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Approach** | Map-based `DynamicDTO` — zero entity classes | JPA/Hibernate with `@Entity` per table |
| **Why** | Customers create new BOs at runtime (via DDL endpoints). Static entities would require code changes + redeployment for every new table. |
| **Trade-off** | No compile-time type safety. `dto.get("WELL_ID")` returns `Object` — typos and type mismatches are runtime errors. |
| **Mitigation** | Constants in `DSPDMConstants.BoAttrName` for frequently-used columns. Metadata validation at startup. |

### Decision 2: Metadata-Driven SQL Generation (No ORM)

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Approach** | Build SQL strings at runtime from `BUSINESS_OBJECT` + `BUSINESS_OBJECT_ATTR` metadata tables | Hibernate HQL/Criteria, MyBatis |
| **Why** | When the data model changes (new BO, new attribute), only metadata rows change — zero code changes. The DAO factory rebuilds from metadata. |
| **Trade-off** | Complex SQL builder code (~6000+ lines in `AbstractDynamicDAOImplFor*` chain). Harder to debug than declarative ORM. |
| **Mitigation** | SQL logging (`showSQLStats`), `ExecutionContext` tracks all generated SQL for debugging. |

### Decision 3: Explicit `ExecutionContext` Passing (Not ThreadLocal)

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Approach** | `ExecutionContext` passed as explicit parameter to every method | ThreadLocal (Spring Security style), CDI `@RequestScoped` |
| **Why** | Makes dependencies explicit — no hidden global state. Easier to unit test (just construct a context). Works across thread boundaries. |
| **Trade-off** | Verbose method signatures — every method has `(..., ExecutionContext executionContext)` as last parameter. |

### Decision 4: Multi-Tenant via Row-Level Security

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Approach** | Single shared database, tenant ID set on every Postgres connection via `set_config('app.current_tenant', tenantId)`, row-level security (RLS) policies filter data | Separate database per tenant |
| **Why** | Cost-efficient (one DB), no need to maintain N databases |
| **Trade-off** | A bug in RLS setup could leak data between tenants. Noisy-neighbor performance risk. |

### Decision 5: Teiid Virtual Database Layer

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Approach** | Teiid VDB with virtual views + UDFs sitting between the app and PostgreSQL | Direct JDBC to Postgres |
| **Why** | Provides a canonical data model (100+ views), federation capability, Java UDFs for complex operations (blob manipulation, unit conversion, CRS transformation) |
| **Trade-off** | Extra network hop, additional infrastructure (WildFly + Teiid), debugging complexity |

### Decision 6: Deep Inheritance Chain for DAOs

```
BaseDAO → MetadataCacheDAO → BaseDynamicDAO → ForRead → ForJoiningRead
    → ForInsert → ForUpdate → ForDelete → ForStructureChange
        → ForReverseEngineer → AbstractDynamicDAOImpl → DynamicDAO
```

| Aspect | Choice Made | Alternative |
|--------|-------------|-------------|
| **Why** | Each level adds exactly one capability. Clean separation of concerns — read logic doesn't know about insert logic. | Composition over inheritance (strategy objects for read/write/delete) |
| **Trade-off** | 10-level inheritance depth. Hard to navigate. Changes at base levels ripple through entire chain. |

### Decision 7: Three-Class Pattern for Delegates

```
IXxxDelegate        ← interface (contract)
XxxDelegate         ← outer singleton (exception handling + transactions)
XxxDelegateImpl     ← inner singleton (pure business logic)
```

| **Why** | Separation of cross-cutting concerns from business logic |
|---------|--------------------------------------------------------|
| **Outer** handles | Exception wrapping, transaction begin/commit/rollback |
| **Inner** handles | Pure business logic, DAO calls |
| **Result** | Inner class is easy to test in isolation |

## **When DynamicDAOFactory.refreshMetadata() is Called**

The `DynamicDAOFactory.refreshMetadata()` method is a **critical system operation** that rebuilds the entire DAO registry from database metadata. Here's when and how it works:

### **Automatic Triggers**

| **Event** | **When Called** | **Why** |
|-----------|-----------------|---------|
| **System Startup** | Application boot (DynamicDAOFactory initialization) | Load ALL metadata into memory for first time |
| **New Table Added** | After `POST /api/v1/metadata/addBusinessObjects` | Register new BO in DAO factory |
| **Table Dropped** | After `POST /api/v1/metadata/dropBusinessObjects` | Remove BO from DAO factory |
| **New Column Added** | After `POST /api/v1/metadata/addBusinessObjectAttrs` | Update column registry for existing BO |
| **Column Dropped** | After `POST /api/v1/metadata/dropBusinessObjectAttrs` | Remove column from BO registry |
| **Constraints Added** | After `POST /api/v1/metadata/addUniqueConstraints` | Update constraint metadata |
| **Relationships Added** | After `POST /api/v1/metadata/addRelationships` | Update foreign key relationships |

### **Manual Triggers**

| **REST Endpoint** | **Purpose** | **Use Case** |
|-------------------|-------------|--------------|
| `GET /api/v1/refreshMetadata` | Force refresh all metadata | After direct DB schema changes |
| `GET /api/v1/validateMetadata` | Validate + refresh if needed | Health check, troubleshooting |
| Admin console action | UI-triggered refresh | Operations team maintenance |

### **Complete Refresh Process**

```java
// Real DSPDM code pattern for metadata refresh
public synchronized void refreshMetadata(ExecutionContext ctx) {
    try {
        // 1. CLEAR existing DAO registry
        daoMap.clear();
        
        // 2. RE-READ all metadata from service database
        List<DynamicDTO> businessObjects = readAllBusinessObjects();
        List<DynamicDTO> businessObjectAttrs = readAllBusinessObjectAttrs();
        List<DynamicDTO> relationships = readAllRelationships();
        List<DynamicDTO> constraints = readAllConstraints();
        
        // 3. REBUILD DAO instances for each BO
        for (DynamicDTO boDTO : businessObjects) {
            String boName = boDTO.getString("BO_NAME");
            
            // Create new DAO with fresh metadata
            DynamicDAO dao = new DynamicDAO(boName);
            
            // Register columns from BUSINESS_OBJECT_ATTR
            List<DynamicDTO> columns = filterColumnsByBO(businessObjectAttrs, boName);
            dao.registerColumns(columns);
            
            // Register constraints
            List<DynamicDTO> boConstraints = filterConstraintsByBO(constraints, boName);
            dao.registerConstraints(boConstraints);
            
            // Register relationships (foreign keys)
            List<DynamicDTO> boRelationships = filterRelationshipsByBO(relationships, boName);
            dao.registerRelationships(boRelationships);
            
            // Add to registry
            daoMap.put(boName.toUpperCase(), dao);
        }
        
        // 4. MAKE metadata immutable after refresh
        makeAllMetadataListsUnmodifiable();
        
        // 5. LOG success
        logger.info("Metadata refreshed: {} BOs, {} total columns", 
                   businessObjects.size(), businessObjectAttrs.size());
                   
    } catch (Exception e) {
        // 6. ROLLBACK on failure - keep old metadata
        logger.error("Metadata refresh failed, keeping existing registry", e);
        throw new DSPDMException("Failed to refresh metadata: {}", e.getMessage());
    }
}
```

### **What Gets Refreshed**

| **Metadata Component** | **Source Table** | **Impact After Refresh** |
|------------------------|------------------|---------------------------|
| **Business Objects** | `BUSINESS_OBJECT` | New/removed tables available in DAO registry |
| **Column Definitions** | `BUSINESS_OBJECT_ATTR` | New columns, data types, nullable flags |
| **Primary Keys** | `BUSINESS_OBJECT_ATTR.IS_PRIMARY_KEY=true` | PK validation in save operations |
| **Mandatory Fields** | `BUSINESS_OBJECT_ATTR.IS_MANDATORY=true` | Validation rules in insert/update |
| **Data Types** | `BUSINESS_OBJECT_ATTR.DATA_TYPE` | SQL generation, type casting |
| **Relationships** | `BUSINESS_OBJECT_RELATIONSHIP` | Foreign key validation, JOIN queries |
| **Unique Constraints** | `BUS_OBJ_ATTR_UNIQ_CONSTRAINTS` | Duplicate validation |
| **Search Indexes** | `BUS_OBJ_ATTR_SEARCH_INDEX` | Full-text search capability |

### **Memory Impact**

```
Before Refresh:  Old DAO registry in memory (2-5 MB typically)
During Refresh:  Both old + new registries temporarily (4-10 MB)
After Refresh:   New registry only, old GC'd (2-5 MB)
```

## **When Redis Cache is Updated in DSPDM**

Redis serves as DSPDM's **distributed cache layer** for performance optimization. Here's when and how the Redis cache gets updated:

### **Cache Update Triggers**

| **Event Type** | **What Gets Updated** | **Update Method** | **Frequency** |
|----------------|----------------------|-------------------|---------------|
| **Reference Data Change** | `R_WELL_TYPE`, `R_FIELD`, `R_STATUS` lookup tables | Cache-aside pattern | On demand |
| **User Permission Change** | Keycloak roles → BO permissions mapping | Manual invalidation | When roles change |
| **Metadata Refresh** | BO structure, column definitions | Manual clear + reload | After schema changes |
| **Heartbeat Check** | Cache health status, staleness detection | Background process | Every 30 seconds |

### **Cache-Aside Pattern (Primary Method)**

```java
// Real DSPDM Redis cache-aside implementation
public List<DynamicDTO> getReferenceData(String boName, ExecutionContext ctx) {
    // 1. CHECK Redis cache first
    String cacheKey = buildTenantIsolatedKey(boName, ctx.getTenantId());
    List<DynamicDTO> cachedData = DSPDMRedisCacheUtils.get(cacheKey);
    
    if (cachedData != null) {
        // Cache HIT - return immediately
        logger.debug("Cache hit for reference data: {}", boName);
        return cachedData;
    }
    
    // 2. Cache MISS - fetch from database
    logger.debug("Cache miss for reference data: {}, fetching from DB", boName);
    List<DynamicDTO> dbData = dynamicDAO.read(buildReferenceQuery(boName), ctx);
    
    // 3. POPULATE cache for future requests
    DSPDMRedisCacheUtils.set(cacheKey, dbData, REFERENCE_DATA_TTL);
    
    return dbData;
}
```

| Reference record modified | Same cache key cleared | Well status name changed → cache invalidated |

## **How Lazy Loading Works in DSPDM**

DSPDM implements **relationship-based lazy loading** to optimize performance by fetching related data only when explicitly requested, rather than loading everything upfront. This approach avoids N+1 queries through intelligent batching and caching strategies.

### **Lazy Loading Architecture**

```java
// Real DSPDM lazy loading pattern in BusinessObjectReadDelegateImpl
@Override
public Map<String, Object> readComplex(BusinessObjectInfo businessObjectInfo, ExecutionContext executionContext) {
    // 1. ALWAYS load main business object data first
    List<DynamicDTO> boList = readSimple(businessObjectInfo, executionContext);
    
    // 2. LAZY LOAD parent relationships ONLY if requested
    if (CollectionUtils.hasValue(businessObjectInfo.getParentBONamesToRead())) {
        // Load metadata on demand if not already available
        if (metadataList == null) {
            metadataList = MetadataBOAttrReadDelegateImpl.getInstance().read(
                businessObjectInfo.getBusinessObjectType(), executionContext);
        }
        List<DynamicDTO> referenceMetadataList = getReferenceMetadataList(
            metadataList, businessObjectInfo.getParentBONamesToRead(), executionContext);
        fillRelatedDetails(businessObjectInfo, boList, referenceMetadataList, 1, 
                          new ArrayList<String>(), executionContext);
    }
    
    // 3. LAZY LOAD child relationships ONLY if requested
    if (CollectionUtils.hasValue(businessObjectInfo.getChildBONamesToRead())) {
        fillRelatedChildrenData(businessObjectInfo, boList, 
            DynamicDAOFactory.getInstance(executionContext)
                .getDynamicDAO(businessObjectInfo.getBusinessObjectType(), executionContext)
                .readMetadataRelationships(executionContext), 
            1, new ArrayList<>(), executionContext);
    }
    
    return map;
}
```

### **Lazy Loading Triggers**

| **Trigger** | **Implementation** | **Performance Impact** |
|-------------|-------------------|------------------------|
| **Parent BO Request** | `businessObjectInfo.getParentBONamesToRead()` | Single query per parent BO type |
| **Child BO Request** | `businessObjectInfo.getChildBONamesToRead()` | Batched by foreign key relationships |
| **Reference Data** | Cache-aside pattern with Redis | Cache hit = 1ms, miss = 50-100ms |
| **Metadata Loading** | On-demand metadata fetch only when needed | Prevents startup overhead |

### **N+1 Query Prevention Strategies**

#### **1. Relationship Batching**
```java
// Real implementation from fillRelatedDetails()
private void fillRelatedDetails(BusinessObjectInfo businessObjectInfo, 
                               List<DynamicDTO> businessObjectsList, 
                               List<DynamicDTO> childReferenceMetadataList, 
                               int currentHierarchyDepth, List<String> messages, 
                               ExecutionContext executionContext) {
    
    // Process ALL objects in current batch together - not one by one
    for (DynamicDTO businessObjectDTO : businessObjectsList) {
        for (DynamicDTO metadataDTO : childReferenceMetadataList) {
            // Extract foreign key values for batch processing
            childFKBOAttrValue = businessObjectDTO.get(childFKBOAttrName);
            
            if (childFKBOAttrValue != null) {
                // Create filter for batch lookup
                BusinessObjectInfo parentBusinessObjectInfo = new BusinessObjectInfo(parentBOName, executionContext);
                parentBusinessObjectInfo.addFilter(parentPKBOAttrName, childFKBOAttrValue);
                
                // SINGLE query gets all parent records for this FK value
                parentBOList = readSimple(parentBusinessObjectInfo, executionContext);
            }
        }
    }
}
```

#### **2. Cache-Aside Pattern for Reference Data**
```java
// Real DSPDM cache-aside implementation from ReferenceDataReadDelegateImpl
public Map<String, PagedList<DynamicDTO>> read(String boName, ExecutionContext executionContext) {
    // 1. CHECK cache first - prevents repeated DB hits
    String cacheKey = DSPDMRedisCacheUtils.buildTenantIsolatedKey(boName, executionContext.getTenantId());
    Map<String, PagedList<DynamicDTO>> cachedData = DSPDMRedisCacheUtils.get(cacheKey);
    
    if (cachedData != null) {
        logger.debug("Reference data cache HIT for BO: {}", boName);
        return cachedData; // Return immediately - no DB query needed
    }
    
    // 2. Cache MISS - load from database once
    logger.debug("Reference data cache MISS for BO: {}, loading from database", boName);
    Map<String, PagedList<DynamicDTO>> dbData = readFromCurrent(businessObjectInfo, boName, metadataList, executionContext);
    
    // 3. POPULATE cache for future requests
    DSPDMRedisCacheUtils.set(cacheKey, dbData, ConfigProperties.getInstance().reference_data_cache_ttl.getIntegerValue());
    
    return dbData;
}
```

#### **3. Batch Metadata Loading**
```java
// Metadata loaded in batches, not per-field
private List<DynamicDTO> getReferenceMetadataList(List<DynamicDTO> metadataList, 
                                                  List<String> filterReferenceBONames, 
                                                  ExecutionContext executionContext) {
    // Load metadata for ALL requested BOs in single operation
    List<DynamicDTO> referenceList = new ArrayList<>();
    List<String> alreadyAddedBONames = new ArrayList<>();
    
    for (DynamicDTO metadataDTO : metadataList) {
        String referenceBOName = (String) metadataDTO.get(DSPDMConstants.BoAttrName.REFERENCE_BO_NAME);
        
        // Batch processing - avoid duplicate metadata reads
        if ((StringUtils.hasValue(referenceBOName)) && 
            (!alreadyAddedBONames.contains(referenceBOName)) && 
            CollectionUtils.containsIgnoreCase(filterReferenceBONames, referenceBOName)) {
            
            referenceList.add(metadataDTO);
            alreadyAddedBONames.add(referenceBOName); // Prevent duplicate loading
        }
    }
    return referenceList;
}
```

```java
                                                         ResultSet.TYPE_FORWARD_ONLY,

    // 4. EXECUTE query with controlled resource usage
    ResultSet rs = pstmt.executeQuery();

    closeConnection(boType, connection, executionContext);
}
```

```java
            case "INTEGER":
                int intVal = rs.getInt(columnName);
                return rs.wasNull() ? null : intVal;
                
            case "VARCHAR":
            case "CHAR":
            case "TEXT":
                return rs.getString(columnName);
                
            case "TIMESTAMP":
                return rs.getTimestamp(columnName);
                
            case "DATE":
                return rs.getDate(columnName);
                
            case "DECIMAL":
            case "NUMERIC":

                return blob != null ? blob.getBytes(1, (int) blob.length()) : null;
```

```java
public String buildSelectQuery(BusinessObjectInfo boInfo) {
    StringBuilder sql = new StringBuilder("SELECT ");
    
    // Get column list from metadata registry
    List<String> selectColumns = boInfo.getSelectList();
    if (selectColumns == null || selectColumns.isEmpty()) {
        // Default: all active columns from metadata
        selectColumns = columnRegistry.keySet().stream()
            .filter(this::isActiveColumn)
            .collect(Collectors.toList());
    }
    
    // Build column list with proper data type handling
    List<String> sqlColumns = new ArrayList<>();
    for (String boAttrName : selectColumns) {
        DynamicDTO columnDef = columnRegistry.get(boAttrName);
        if (columnDef != null) {
            String physicalColumn = columnDef.getString("PHYSICAL_COLUMN_NAME");
            String dataType = columnDef.getString("DATA_TYPE");
            
            // Add type conversion for specific databases if needed
            String columnSQL = buildColumnSelect(physicalColumn, dataType, boAttrName);
            sqlColumns.add(columnSQL);
        }
    }
    
    sql.append(String.join(", ", sqlColumns));
    sql.append(" FROM ").append(getTableName());
    
    // Add WHERE, ORDER BY, etc.
    appendWhereClause(sql, boInfo);
    appendOrderByClause(sql, boInfo);
    appendPaginationClause(sql, boInfo);

private String buildColumnSelect(String physicalColumn, String dataType, String alias) {
    // Handle special data type conversions
    switch (dataType.toUpperCase()) {
        case "TIMESTAMP":
            // Ensure proper timezone handling
            return String.format("CAST(%s AS TIMESTAMP) AS %s", physicalColumn, alias);
            
        case "DECIMAL":
        case "NUMERIC":
            // Preserve precision
            return String.format("CAST(%s AS DECIMAL) AS %s", physicalColumn, alias);
            
        default:
            return String.format("%s AS %s", physicalColumn, alias);
    }
}
```

```java
// Validation based on metadata
public void validateFieldValue(String boAttrName, Object value) {
    DynamicDTO columnDef = columnRegistry.get(boAttrName);
    if (columnDef == null) {
        throw new DSPDMValidationException("Unknown field: {}", boAttrName);
    }
    
    String dataType = columnDef.getString("DATA_TYPE");
    Boolean isMandatory = columnDef.getBoolean("IS_MANDATORY");
    Integer dataLength = columnDef.getInteger("DATA_LENGTH");
    
    // Mandatory field validation
    if (isMandatory && (value == null || "".equals(value))) {
        throw new DSPDMValidationException("Field {} is mandatory", boAttrName);
    }
    
    if (value == null) return; // null is valid for optional fields
    
    // Data type validation
    switch (dataType.toUpperCase()) {
        case "VARCHAR":
        case "CHAR":
            if (!(value instanceof String)) {
                throw new DSPDMValidationException("Field {} must be string", boAttrName);
            }
            if (dataLength != null && value.toString().length() > dataLength) {
                throw new DSPDMValidationException(
                    "Field {} exceeds max length {}", boAttrName, dataLength);
            }
            break;
            
        case "BIGINT":
            if (!(value instanceof Number)) {
                throw new DSPDMValidationException("Field {} must be number", boAttrName);
            }
            break;
            
        case "TIMESTAMP":
            if (!(value instanceof Timestamp) && !(value instanceof String)) {
                throw new DSPDMValidationException(
                    "Field {} must be timestamp or ISO string", boAttrName);
            }
            // Try to parse if string
            if (value instanceof String) {
                try {
                    Timestamp.valueOf((String) value);
                } catch (IllegalArgumentException e) {
                    throw new DSPDMValidationException(
                        "Field {} has invalid timestamp format: {}", boAttrName, value);
                }
            }
            break;
    }
}
```

```java
public class ReportingEntityManager {

    public List<DynamicDTO> getProductionByEntity(Long reportingEntityId, 
                                                 Date startDate, Date endDate,
                                                 ExecutionContext ctx) {

        List<Long> childEntityIds = getChildEntities(reportingEntityId);
        
        // 2. OPTIMIZE query by entity level
        if (childEntityIds.size() > 1000) {

            return getMonthlyAggregatedData(childEntityIds, startDate, endDate, ctx);

            // Use daily data for smaller hierarchies
            return getDailyProductionData(childEntityIds, startDate, endDate, ctx);

    private List<Long> getChildEntities(Long reportingEntityId) {
        // Check cache first
        if (entityHierarchyCache.containsKey(reportingEntityId)) {
            return entityHierarchyCache.get(reportingEntityId);

        // Build hierarchy from database
        String hierarchySQL = """
            WITH RECURSIVE entity_tree AS (
                SELECT reporting_entity_id, parent_entity_id, entity_level, hierarchy_path
                FROM reporting_entity_hierarchy 
                WHERE reporting_entity_id = ? AND is_active = true
                
                UNION ALL
                
                SELECT h.reporting_entity_id, h.parent_entity_id, h.entity_level, h.hierarchy_path
                FROM reporting_entity_hierarchy h
                INNER JOIN entity_tree et ON h.parent_entity_id = et.reporting_entity_id
                WHERE h.is_active = true
            )
            SELECT reporting_entity_id FROM entity_tree ORDER BY entity_level, reporting_entity_id
            """;
        
        List<Long> childIds = executeHierarchyQuery(hierarchySQL, reportingEntityId);
        
        // Cache result for future queries
        entityHierarchyCache.put(reportingEntityId, childIds);
        
        return childIds;
    }
```

```java
public class DateRangeOptimizer {
    
    public String buildOptimizedDateQuery(Date startDate, Date endDate, List<Long> entityIds) {
        
        long daysBetween = ChronoUnit.DAYS.between(startDate.toInstant(), endDate.toInstant());
        
        if (daysBetween <= 31) {
            // Short range - use daily table with date index
            return buildDailyRangeQuery(startDate, endDate, entityIds);
            
        } else if (daysBetween <= 365) {
            // Medium range - use monthly aggregates where possible
            return buildMixedRangeQuery(startDate, endDate, entityIds);
            
        } else {
            // Long range - use yearly aggregates + materialized views
            return buildYearlyRangeQuery(startDate, endDate, entityIds);
        }
    }
    
    private String buildMixedRangeQuery(Date startDate, Date endDate, List<Long> entityIds) {
        
        // Smart approach: use monthly aggregates for complete months, daily for partial months
        return """
            WITH date_ranges AS (
                SELECT 
                    ? as query_start_date,
                    ? as query_end_date,
                    DATE_TRUNC('month', ?) as start_month,
                    DATE_TRUNC('month', ?) as end_month
            ),
            complete_months AS (
                SELECT production_month, SUM(total_oil_volume) as oil_volume,
                       SUM(total_gas_volume) as gas_volume, SUM(total_water_volume) as water_volume
                FROM well_vol_monthly wvm, date_ranges dr
                WHERE wvm.reporting_entity_id = ANY(?)
                    AND wvm.production_month > dr.start_month 
                    AND wvm.production_month < dr.end_month
                GROUP BY production_month
            ),
            partial_days AS (
                SELECT production_date, SUM(oil_volume) as oil_volume,
                       SUM(gas_volume) as gas_volume, SUM(water_volume) as water_volume
                FROM well_vol_daily wvd, date_ranges dr
                WHERE wvd.reporting_entity_id = ANY(?)
                    AND ((wvd.production_date >= dr.query_start_date 
                          AND wvd.production_date < dr.start_month + INTERVAL '1 month')
                         OR (wvd.production_date >= dr.end_month 
                             AND wvd.production_date <= dr.query_end_date))
                    AND wvd.status = 'ACTIVE'
                GROUP BY production_date
            )
            SELECT * FROM complete_months 
            UNION ALL 
            SELECT * FROM partial_days
            ORDER BY 1
            """;
    }
}
```

```java
// DSPDM automatically reorders filters for optimal performance
public class FilterOptimizer {
    
    public String optimizeFilterOrder(List<CriteriaFilter> filters) {
        
        // 1. SORT filters by selectivity (most selective first)
        filters.sort((f1, f2) -> {
            double selectivity1 = calculateSelectivity(f1);
            double selectivity2 = calculateSelectivity(f2);
            return Double.compare(selectivity2, selectivity1); // Descending
        });
        
        // 2. BUILD optimized WHERE clause
        StringBuilder whereClause = new StringBuilder("WHERE ");
        
        for (int i = 0; i < filters.size(); i++) {
            CriteriaFilter filter = filters.get(i);
            
            if (i > 0) whereClause.append(" AND ");
            
            // Place most selective filters first for short-circuit evaluation
            whereClause.append(buildFilterCondition(filter));
        }
        
        return whereClause.toString();
    }
    
    private double calculateSelectivity(CriteriaFilter filter) {
        
        switch (filter.getOperator()) {
            case "equals":
                if (filter.getBoAttrName().endsWith("_ID")) {
                    return 0.999; // ID filters are highly selective
                }
                return 0.8; // Regular equals
                
            case "in": 
                List<?> values = (List<?>) filter.getValue();
                return Math.max(0.1, 1.0 - (values.size() / 1000.0)); // Less selective with more values
                
            case "between":
                if (filter.getBoAttrName().contains("DATE")) {
                    return calculateDateRangeSelectivity(filter);
                }
                return 0.5; // Moderate selectivity
                
            case "like":
                return 0.3; // Generally less selective
                
            case "isNull":
            case "isNotNull":
                return 0.2; // Low selectivity
                
            default:
                return 0.5; // Default moderate selectivity
        }
    }
}
```

```java
// Real DSPDM pattern for automatic table selection
public class ProductionTableRouter {
    
    public TableSelectionStrategy selectOptimalTable(ProductionQueryRequest request) {
        
        // 1. ANALYZE query characteristics
        QueryCharacteristics characteristics = analyzeQuery(request);
        
        // 2. DETERMINE optimal table based on multiple factors
        if (characteristics.requiresRealTimeData()) {
            return useDetailedTable(request, characteristics);
            
        } else if (characteristics.isSimpleEntityQuery()) {
            return useAggregatedTable(request, characteristics);
            
        } else if (characteristics.isComplexHierarchyQuery()) {
            return useMaterializedView(request, characteristics);
            
        } else {
            return useHybridApproach(request, characteristics);
        }
    }
    
    private QueryCharacteristics analyzeQuery(ProductionQueryRequest request) {
        
        QueryCharacteristics characteristics = new QueryCharacteristics();
        
        // Date range analysis
        long daysBetween = ChronoUnit.DAYS.between(
            request.getStartDate().toInstant(), 
            request.getEndDate().toInstant()
        );
        characteristics.setDateRangeType(
            daysBetween <= 31 ? "DAILY" : 
            daysBetween <= 365 ? "MONTHLY" : "YEARLY"
        );
        
        // Entity complexity analysis
        int entityCount = request.getEntityIds().size();
        String entityType = request.getEntityType();
        characteristics.setEntityComplexity(
            entityCount == 1 && "WELL".equals(entityType) ? "SIMPLE" :
            entityCount <= 100 ? "MEDIUM" : "COMPLEX"
        );
        
        // Aggregation requirements
        boolean needsGrouping = !request.getGroupByFields().isEmpty();
        boolean needsCalculations = !request.getCalculatedFields().isEmpty();
        characteristics.setAggregationLevel(
            needsGrouping || needsCalculations ? "HIGH" : "LOW"
        );
        
        // Real-time requirements
        boolean realTimeRequired = request.isRealTimeRequired() || daysBetween <= 1;
        characteristics.setRealTimeRequired(realTimeRequired);
        
        return characteristics;
    }
}
```

```java
                searchJson.addProperty(fieldName.toLowerCase(), searchValue);
            }
        }

        return gson.toJson(businessObject.getDataMap());
    }

        return value.toLowerCase()            // Case insensitive
                   .trim()                    // Remove whitespace
                   .replaceAll("[^a-zA-Z0-9\\s]", "");  // Remove special chars
```

```java
// Real DSPDM search index update flow from SearchHelper.java
public static SaveResultDTO createSearchIndexForSavedOrUpdatedRecord(String boName,
                                                                    List<DynamicDTO> currentDTOList,
                                                                    IDynamicReadService dynamicReadService,
                                                                    IDynamicWriteService dynamicWriteService,
                                                                    ExecutionContext executionContext) {
    
    // 1. EXTRACT primary keys of updated records only
    String primaryKeyAttrName = getPrimaryKeyColumn(boName);
    List<Object> currentPrimaryKeyList = CollectionUtils.getValuesFromList(currentDTOList, primaryKeyAttrName);
    
    // 2. DELETE existing BO_SEARCH entries for these specific records
    BusinessObjectInfo boSearchBusinessObjectInfo = new BusinessObjectInfo(DSPDMConstants.BoName.BO_SEARCH, executionContext);
    boSearchBusinessObjectInfo.addFilter(DSPDMConstants.BoAttrName.BO_NAME, Operator.EQUALS, boName);
    boSearchBusinessObjectInfo.addFilter(DSPDMConstants.BoAttrName.PK_BO_ATTR_VALUES, Operator.IN, currentPrimaryKeyList);
    
    List<DynamicDTO> existingSearchEntries = dynamicReadService.readSimple(boSearchBusinessObjectInfo, executionContext);
    if (CollectionUtils.hasValue(existingSearchEntries)) {
        // DELETE only the affected search index records
        DynamicDAOFactory.getInstance(executionContext).getDynamicDAO(DSPDMConstants.BoName.BO_SEARCH, executionContext)
            .delete(existingSearchEntries, executionContext);
    }
    
    // 3. RE-READ updated records from main table with fresh data
    BusinessObjectInfo mainBoInfo = new BusinessObjectInfo(boName, executionContext);
    mainBoInfo.addFilter(primaryKeyAttrName, Operator.IN, currentPrimaryKeyList);
    List<DynamicDTO> freshRecords = dynamicReadService.readSimple(mainBoInfo, executionContext);
    
    // 4. CREATE new BO_SEARCH entries with updated JSONB
    List<DynamicDTO> newSearchIndexes = new ArrayList<>();
    for (DynamicDTO freshRecord : freshRecords) {
        DynamicDTO searchIndexEntry = createBOSearchDTO(boName, freshRecord, executionContext);
        // This includes both SEARCH_JSONB and OBJECT_JSONB with latest data
        newSearchIndexes.add(searchIndexEntry);
    }
    
    // 5. INSERT new search index entries
    return DynamicDAOFactory.getInstance(executionContext).getDynamicDAO(DSPDMConstants.BoName.BO_SEARCH, executionContext)
        .saveOrUpdate(newSearchIndexes, executionContext);
            
}

// KEY POINTS:
// - Only processes records that were actually updated (efficient)
// - DELETE + INSERT pattern ensures GIN index gets fresh JSONB data
// - No partial updates - complete JSONB regeneration for changed records
// - Background processing prevents user interface blocking
```

```java
// Complete metadata refresh when cache miss occurs
public class DynamicDAOFactory {
    
    public synchronized boolean refreshMetadata() {
        
        logger.info("Starting metadata refresh due to cache miss...");
        long startTime = System.nanoTime();
        
        try {
            // 1. CLEAR existing metadata cache
            clearAllCaches();
            
            // 2. RELOAD business objects from database
            Map<String, BusinessObjectMetadata> newMetadata = loadBusinessObjectMetadata();
            
            // 3. REBUILD DAO registry
            Map<String, IDynamicDAO> newDAORegistry = new HashMap<>();
            
            for (Map.Entry<String, BusinessObjectMetadata> entry : newMetadata.entrySet()) {
                String boName = entry.getKey();
                BusinessObjectMetadata metadata = entry.getValue();
                
                try {
                    // Create new DAO for each business object
                    IDynamicDAO dao = createDynamicDAO(metadata);
                    newDAORegistry.put(boName, dao);
                    
                    logger.debug("Created DAO for business object: {}", boName);
                    
                } catch (Exception e) {
                    logger.error("Failed to create DAO for {}: {}", boName, e.getMessage());
                    // Continue processing other tables
                }
            }
            
            // 4. ATOMIC replacement of old registry
            this.daoRegistry = Collections.unmodifiableMap(newDAORegistry);
            this.metadataRegistry = Collections.unmodifiableMap(newMetadata);
            
            long executionTime = TimeUnit.NANOSECONDS.toMillis(System.nanoTime() - startTime);
            logger.info("Metadata refresh completed successfully in {}ms. Loaded {} business objects.", 
                       executionTime, newDAORegistry.size());
            
            return true;
            
        } catch (Exception e) {
            logger.error("Metadata refresh failed: {}", e.getMessage());
            return false;
        }
    }
    
    private Map<String, BusinessObjectMetadata> loadBusinessObjectMetadata() {
        
        Map<String, BusinessObjectMetadata> metadata = new HashMap<>();
        
        // Load business objects
        String boSQL = """
            SELECT bo_name, table_name, schema_name, is_active, pk_bo_attr_names
            FROM business_object 
            WHERE is_active = true
            ORDER BY bo_name
            """;
            
        List<DynamicDTO> businessObjects = executeMetadataQuery(boSQL);
        
        for (DynamicDTO bo : businessObjects) {
            String boName = bo.getString("BO_NAME");
            
            // Load attributes for each business object
            String attrSQL = """
                SELECT bo_attr_name, physical_column_name, data_type, data_length,
                       is_mandatory, default_value, column_order
                FROM business_object_attr 
                WHERE bo_name = ? AND is_active = true
                ORDER BY column_order
                """;
                
            List<DynamicDTO> attributes = executeMetadataQuery(attrSQL, boName);
            
            // Create metadata object
            BusinessObjectMetadata boMetadata = new BusinessObjectMetadata(
                boName,
                bo.getString("TABLE_NAME"),
                bo.getString("SCHEMA_NAME"),
                attributes
            );
            
            metadata.put(boName, boMetadata);
        }
        
        return metadata;
    }
```

```java
    public SaveResultDTO save(List<DynamicDTO> businessObjects,

        if (businessObjects == null || businessObjects.isEmpty()) {
            return new SaveResultDTO();
        }
        
        // Group by business object type for batch processing
        Map<String, List<DynamicDTO>> groupedObjects = businessObjects.stream()
            .collect(Collectors.groupingBy(DynamicDTO::getBoName));
        
        SaveResultDTO aggregatedResult = new SaveResultDTO();
        
        for (Map.Entry<String, List<DynamicDTO>> entry : groupedObjects.entrySet()) {
            String boName = entry.getKey();
            List<DynamicDTO> objects = entry.getValue();
            
            try {
                // 1. ATTEMPT to get DAO (with cache miss handling)
                IDynamicDAO dynamicDAO = getDAOWithFallback(boName, executionContext);
                
                // 2. VALIDATE objects against metadata
                validateObjectsAgainstMetadata(objects, boName, executionContext);
                
                // 3. EXECUTE save operation
                SaveResultDTO result = dynamicDAO.save(objects, executionContext);
                aggregatedResult.merge(result);
                
                // 4. UPDATE search indexes asynchronously
                scheduleSearchIndexUpdate(objects, executionContext);
                
            } catch (Exception e) {
                logger.error("Save operation failed for {}: {}", boName, e.getMessage());
                aggregatedResult.addError(boName, e.getMessage());
            }
        }
        
        return aggregatedResult;
    }
    
    private IDynamicDAO getDAOWithFallback(String boName, ExecutionContext executionContext) {
        
        // First attempt - check cache
        IDynamicDAO dao = DynamicDAOFactory.getInstance().getDAO(boName);
        
        if (dao != null) {
            return dao; // Cache HIT
        }
        
        // Cache MISS - check if table exists in database
        logger.warn("DAO cache miss for {}, checking database...", boName);
        
        if (tableExistsInDatabase(boName, executionContext)) {
            
            // Table exists - refresh metadata cache
            boolean refreshed = DynamicDAOFactory.getInstance().refreshMetadata();
            
            if (refreshed) {
                dao = DynamicDAOFactory.getInstance().getDAO(boName);
                
                if (dao != null) {
                    logger.info("Successfully loaded DAO for new table: {}", boName);
                    return dao;
                }
            }
        }
        
        // Table doesn't exist - create new table dynamically
        return createNewTableAndDAO(boName, executionContext);
    }
```

```java
public class MSMHelper {
    
    public static Double convertValue(String fromUnit, String toUnit, Double value) {
        
        // 1. CHECK in-memory conversion cache
        String conversionKey = fromUnit + ":" + toUnit;
        ConversionFactor factor = conversionCache.get(conversionKey);
        
        if (factor != null) {
            // Cache HIT - direct conversion
            return value * factor.getMultiplier() + factor.getOffset();
        }
        
        // 2. CACHE MISS - rebuild conversion matrix
        logger.debug("Unit conversion cache miss for {} to {}", fromUnit, toUnit);
        
        synchronized (MSMHelper.class) {
            
            // Double-check pattern
            factor = conversionCache.get(conversionKey);
            if (factor != null) {
                return value * factor.getMultiplier() + factor.getOffset();
            }
            
            // 3. RELOAD unit catalog from database
            boolean reloaded = reloadUnitCatalog();
            
            if (reloaded) {
                factor = conversionCache.get(conversionKey);
                
                if (factor != null) {
                    logger.info("Successfully loaded unit conversion: {} to {}", fromUnit, toUnit);
                    return value * factor.getMultiplier() + factor.getOffset();
                }
            }
        }
        
        // 4. CONVERSION not found - return original value with warning
        logger.warn("Unit conversion not available from {} to {}, returning original value", 
                   fromUnit, toUnit);
        return value;
    }
    
    private static boolean reloadUnitCatalog() {
        
        try {
            logger.info("Reloading unit catalog due to cache miss...");
            
            // Load units from database
            String unitSQL = """
                SELECT unit_code, unit_name, unit_type_code, base_unit_code,
                       conversion_factor, conversion_offset
                FROM unit_catalog 
                WHERE is_active = true
                ORDER BY unit_type_code, unit_code
                """;
                
            List<DynamicDTO> units = executeQuery(unitSQL);
            
            // Rebuild conversion cache
            Map<String, ConversionFactor> newCache = new HashMap<>();
            
            // Build direct conversions
            for (DynamicDTO unit : units) {
                String unitCode = unit.getString("UNIT_CODE");
                String baseUnit = unit.getString("BASE_UNIT_CODE");
                Double factor = unit.getDouble("CONVERSION_FACTOR");
                Double offset = unit.getDouble("CONVERSION_OFFSET");
                
                if (baseUnit != null && factor != null) {
                    // Forward conversion: unit to base
                    newCache.put(unitCode + ":" + baseUnit, 
                               new ConversionFactor(factor, offset != null ? offset : 0.0));
                    
                    // Reverse conversion: base to unit
                    newCache.put(baseUnit + ":" + unitCode, 
                               new ConversionFactor(1.0 / factor, 
                                                   offset != null ? -offset / factor : 0.0));
                }
            }
            
            // Build transitive conversions (A->B->C = A->C)
            buildTransitiveConversions(newCache, units);
            
            // Atomic replacement
            conversionCache = Collections.unmodifiableMap(newCache);
            
            logger.info("Unit catalog reload completed. {} conversions loaded.", newCache.size());
            
            return true;
            
        } catch (Exception e) {
            logger.error("Failed to reload unit catalog: {}", e.getMessage());
            return false;
```

## 10. Real Interview Explanation

> *"I worked on DSPDM — a **metadata-driven data management platform** for the **oil & gas industry**. The key architectural decision was to use **no static entity classes**. Instead, we have a central `DynamicDTO` which is a Map-based DTO extending `TreeMap` with case-insensitive keys. The entire data model — tables, columns, constraints, relationships — is stored in metadata tables in the database itself.*
>
> *At startup, our `DynamicDAOFactory` reads all metadata and creates a DAO per business object with registered columns, data types, and relationships. At runtime, SQL is generated dynamically from this metadata — SELECT, INSERT, UPDATE, DELETE, even DDL for creating new tables. This means customers can add new business objects through the API without any code changes or redeployments.*
>
> *The architecture is layered:*
> - *JAX-RS REST controllers (75+ endpoints)*
> - *Service Facade (zero-logic pass-through)*
> - *Business Delegates (with JTA transactions via Bitronix TM)*
> - *Dynamic DAOs (metadata-driven JDBC)*
> - *PostgreSQL via a Teiid Virtual Database (100+ virtual views, 50+ UDFs)*
>
> *Key technical decisions:*
> - *`ExecutionContext` is passed explicitly (not ThreadLocal) for testability and clarity*
> - *Multi-tenancy via PostgreSQL Row-Level Security (tenant set per connection)*
> - *Three-layer caching: in-memory metadata, Redis for reference data, and unit conversion cache*
> - *WAF bypass tokens for Azure WAF compatibility*
> - *Equipment BOs use an EAV pattern with column-oriented virtual views*
> - *Deep inheritance chain in DAO layer — each level adds exactly one capability (read, insert, update, delete, DDL)*
> - *Three-class pattern for delegates: interface + outer proxy (transactions/exceptions) + inner impl (pure logic)*
>
> *The trade-off is no compile-time type safety — everything is `Object` — but we mitigate that with 2350 lines of constants, metadata validation at startup, and comprehensive input validation at the REST layer."*

## 13. How Redis Is Used

### Overview

Redis is an **optional cache layer** controlled by config property `redis_cache_enabled`. When disabled, all reads fall back to PostgreSQL. Implemented as a Jedis cluster client in `dspdm-redis`.

**Key classes**: `DSPDMRedisCacheClient` (Jedis wrapper), `DSPDMRedisCacheUtils` (serialization), `RedisConfigDelegateImpl` (high-level manager singleton in `dspdm-delegate`).

### 4 Major Use Cases

#### 1. Reference Data Cache (Lookup Tables)
Lookup tables like `R_WELL_TYPE`, `R_STATUS`, `R_FACILITY_TYPE` are cached per BO name + tenant ID.
```
Redis Key: <tenantId>:<boName>   e.g.  dfs-tenant-1:R_WELL_TYPE
Value:      JSON-serialized List<DynamicDTO>
TTL:        cache_ttl_seconds config property
```
On cache miss ? read from PostgreSQL ? write to Redis.

#### 2. Metadata Cache (BO Structure)
BO attribute definitions cached to skip `metadata_db` reads on every API call. Invalidated via `GET /refreshMetadata`.

#### 3. API Permission Cache
User role/permission lookups cached by `<tenantId>:<userName>:<permission>`. Avoids a DB call per request.

#### 4. DAO Factory Instance Cache
`DynamicDAOFactory` caches per-BO DAO configuration in Redis to speed container restart time.

### Flow
```
Request ? RedisConfigDelegateImpl.read(boName)
    redis_cache_enabled == true?
       YES ? DSPDMRedisCacheClient.get(key)
                cache HIT  ? deserialize ? return
                cache MISS ? PostgreSQL read ? Redis.set(key, value, ttl) ? return
       NO  ? PostgreSQL read ? return
```

### Cache Invalidation

| Event | Cache Cleared |
|---|---|
| `GET /redisRefreshAll` | All keys for current tenant |
| `GET /refreshMetadata` | Metadata keys only |
| `POST /metadata/write/*` | Specific BO metadata key |
| App restart | JVM cache cleared; Redis TTL-based expiry continues |

### What is PVS (Product Volume Summary)?

`PRODUCT_VOLUME_SUMMARY` is a **pre-aggregated, denormalized reporting table**. Production volumes are pre-computed and stored by well/facility, flow type, quantity method, date bucket, and product type � avoiding expensive GROUP BY queries at request time.

## 15. Searching Millions of Records � BO_SEARCH

### The BO_SEARCH Table

`BO_SEARCH` is a **flat denormalized search index**. Instead of running `ILIKE` across every column of every BO table, searchable values are pre-indexed here.

```sql
BO_SEARCH (
    BO_SEARCH_ID       SERIAL PRIMARY KEY,
    BO_NAME            VARCHAR,      -- source BO (e.g. WELL, FACILITY)
    PK_BO_ATTR_VALUES  JSONB,        -- primary key values of source record
    SEARCH_VALUES      JSONB         -- all searchable column values
)
```

### Search Operators

| Operator in request | SQL generated | Use case |
|---|---|---|
| `JSONB_FIND_EXACT` | `search_values @> '{"KEY":"value"}'::jsonb` | Exact match |
| `JSONB_FIND_LIKE` | `search_values::text ILIKE '%value%'` | Substring search |

### Blocked Characters
To prevent JSONB injection: `'` `"` `-` `_` `\` `(` `)` `[` `]` `?` `+` `*` � validated in `SearchHelper.validateCriteriaFiltersValues()`.

### Admin Index Endpoints

| Endpoint | Description |

### Which BOs Get Indexed?
Controlled by `search_index_table_list` config. Excluded: metadata tables, spec/meas catalog tables, `BO_SEARCH` itself, BOs in `NO_CHANGE_TRACK_BO_NAMES`.

### Search Index Update After Every Save

After every `saveOrUpdate` or `xmgmt/save`, the search index is updated synchronously:
```java
createSearchIndexForSavedOrUpdatedRecord(saveResultDTO, map, executionContext);
```

**`SearchHelper.createSearchIndexForSavedOrUpdatedRecord()` steps**:
1. Check if BO is in `search_index_table_list` ? if not, skip
2. For each saved/updated record:
   - `DELETE FROM bo_search WHERE bo_name=? AND pk_bo_attr_values=?` (remove stale entries)
   - Re-read the saved record from DB (ensures index reflects committed state)
   - `INSERT INTO bo_search` with JSONB built from all indexable columns
3. For `REPORTING_ENTITY_KIND` variant BOs: indexes each variant separately

**Bulk reindex** (`createSearchIndexForBoNameWithSmallTransactions()`):
- `synchronized(SearchHelper.class)` � prevents concurrent runs
- `isIndexAllInProgress` flag � skips per-BO requests while full reindex is running
- Small transaction batches � avoids OOM and long locks

## 18. Q&A � Critical Flows & Common Questions

### Q: Why no JPA / Hibernate?
**A**: DSPDM uses a **dynamic metadata model** � BO schemas are defined in `metadata_db` at runtime. JPA requires compile-time `@Entity` classes. Since BOs support user-defined custom attributes, static entity classes are impossible. Raw JDBC + `PreparedStatement` + dynamic SQL via `BusinessObjectInfo` is the only viable approach.

---

### Q: How does multi-tenancy work?
**A**: Every request carries a Keycloak JWT. `KeycloakTokenUtils.getTokenInfoFromToken()` extracts `tenantId`. `ExecutionContext` carries `TenantContext` with this ID. Every DAO query routes to the correct PostgreSQL **schema**, and every Redis key is prefixed with `<tenantId>:`.

---

### Q: How are large Excel imports handled?
**A**: `ExcelImportJoinUtil.importFromExcel()` uses Apache POI **streaming SAX mode** (not DOM). Key optimizations:
- Pre-loads all reference data + metadata into `ConcurrentHashMap` before row iteration
- Tracks invalid records in `BulkImportInvalidRecordReasonDTO` without stopping the import
- JSONB mandatory keys validated inline (same logic as `JsonbValidationUtils`)
- Configurable DB batch size for inserts

---

### Q: How does unit conversion work during a save?
**A**: `DTOHelper.recursiveDateUnitConverion()` runs before `saveOrUpdate()`:
1. Reads user's unit template (`USER_UNIT_SETTINGS` + `UNIT_TEMPLATE_DETAILS`) to get target units per attribute
2. `UnitHelper.getUnitNameByMapkey()` looks up the target unit by attribute ID (URL-decodes the value)
3. `UnitUtility.getInstance().valueConversion()` computes converted values
4. On any failure ? `unitConvertException()` throws `RuntimeException`, blocking the save

---

### Q: What is the `%25` bug and how was it fixed?
**A**: Unit symbol `%` appeared as `%25` (URL-encoded) from the DB result map. `UnitUtility` had no mapping for `%25`, throwing `RuntimeException: when converting from %25 to %, the %25 is invalid`. **Fix**: `getUnitNameByMapkey()` now calls `URLDecoder.decode()` on the returned value, with warn-and-fallback if decoding fails.

---

### Q: How does the DAO build dynamic SQL?
**A**: `BusinessObjectInfo` carries the full query descriptor (BO name ? table, WHERE filters, SELECT columns, JOINs, ORDER BY, LIMIT/OFFSET). `AbstractDynamicDAOImplForRead.buildSelectSQL()` traverses it and builds a parameterized SQL string � **no user-value string concatenation**, preventing SQL injection.

---

### Q: Why does `DynamicDTO` extend `TreeMap`?
**A**: `TreeMap` keeps **alphabetical key order**, making JSON responses deterministic and easier to test. It also enables consistent serialization across JVM restarts (HashMap key order is not guaranteed).

---

### Q: What happens during `addBusinessObjects`?
**A**: `POST /metadata/write/addBusinessObjects` ? `AbstractDBStructureChangeDAO`:
1. Validates the new BO definition
2. Executes `CREATE TABLE` DDL built from attribute definitions
3. Inserts rows into `BUSINESS_OBJECT` and `BUSINESS_OBJECT_ATTR`
4. Clears Redis metadata cache
5. Re-loads metadata into `DynamicDAOFactory` � new BO immediately queryable, no restart needed

### 19.9 Security

**Authentication: Keycloak JWT RS256**

`AuthenticationFilter` intercepts every request:
1. Extracts `Authorization: Bearer <token>` header
2. Calls `KeycloakJWTVerifier.verifyToken(token, executionContext)`
3. Decodes JWT header → extracts `kid` (key ID)
4. Fetches RSA public key from Keycloak JWKS endpoint (cached in `ConcurrentHashMap<String, RSAKeyProvider>` by `kid`)
5. Decodes X.509 certificate via `CertificateFactory`
6. Verifies RS256 signature
7. Extracts `tenantId`, `userName`, `roles` claims → populates `ExecutionContext`

**Authorization: Programmatic (no `@RolesAllowed`)**

`BaseController.getCurrentUserExecutionContext(httpHeaders, defaultUser, ApiPermission.EDIT)`:
- Takes `ApiPermission` enum: `OPEN`, `VIEW`, `EDIT`, `DELETE`, `SYSTEM_ADMIN`, `CUSTOM_ATTRIBUTE`
- `AuthorizationHelper.validateApiPermissions()` checks user roles against `ApiPermissionsCache`

**Input sanitization**:
- `DTOHelper.applyWafRule()` called on every incoming string value and key
- `RequestBodyValidationFilter` validates JSON body before resource method
- Base64-encoded payloads decoded in filter (`X-Payload-Encoded-Type: base64`)
- SQL injection: all user values bound via `PreparedStatement.setObject()` — no concatenation

**Multi-tenancy isolation**:
- `AuthenticationFilter` extracts `tenantId` from JWT, validates against loaded tenant list
- All DB queries include tenant-scoped schema routing via `TenantContext` in `ExecutionContext`
- Redis keys prefixed: `<tenantId>:<boName>`

### 19.10 Special / Critical Implementations

#### 1. Fully Dynamic Metadata-Driven System (Most Important)
No static entity classes exist anywhere. Every table's structure (column names, types, lengths, precision, FK references, mandatory flags) is read from `BUSINESS_OBJECT_ATTR` metadata table at runtime. SQL is built completely dynamically. **Adding a new DB column requires zero code change** — only a metadata record insert. This is the core architectural differentiator of DSPDM.

#### 2. Dynamic Unit Conversion Layer (`UnitHelper`)
Applied transparently on both read and write paths. Source unit comes from request; target unit resolved from user's `USER_UNIT_SETTINGS` → fallback to `UNIT_TEMPLATE_DETAILS` → fallback to system default. Special precision handling for `PRESSURE` and `TEMPERATURE` reads a dedicated `PRESSURE_TEMPERATURE` BO. `UnitHelper.unitConvertRowLevel()` handles the PVS case where different rows of the same column carry different units.

#### 3. JSONB Validation with In-Memory Metadata Cache (`JsonbValidationUtils`)
JSONB column values are validated against a reference BO schema before every save. The mandatory field check uses `IS_MANDATORY` metadata flag. Metadata is loaded once per JSONB reference BO and cached in `ConcurrentHashMap` — same pattern as Excel import's `jsonbRefMetadataCache`. This prevents redundant DB reads on high-frequency endpoints.

#### 4. Concurrent Request Processing (`ConcurrentUtils`)
`ConcurrentUtils.concurrentByGroup(supplierList, 10)` processes DTO building in parallel groups of 10. This is critical for bulk save performance — a 500-record payload is split into 50 groups of 10, processed concurrently. Each group submits `Callable<DynamicDTO>` to a thread pool.

#### 5. Excel Bulk Import with Pre-loaded Caches (`ExcelImportJoinUtil`)
Apache POI-based Excel parser with:
- Header row mapped to BO attribute metadata
- Per-row JSONB validation using `jsonbRefMetadataCache` (preloaded before row loop)
- Invalid record tracking via `BulkImportInvalidRecordReasonDTO` — bad rows collected, not aborted
- Unit row parsing for source unit detection
- Configurable batch size for DB inserts

#### 6. `DynamicDTO` as Universal Row Carrier
`DynamicDTO` extends `TreeMap<String, Object>`. No schema-specific subclasses. Every DB row — whether a WELL, R_WELL_TYPE, BUSINESS_OBJECT_ATTR, or BO_SEARCH — is a `DynamicDTO`. `TreeMap` ensures alphabetical key ordering for deterministic JSON serialization.

#### 7. Three-Database Architecture
- `metadata_db`: Stores all BO definitions (`BUSINESS_OBJECT`, `BUSINESS_OBJECT_ATTR`, etc.)
- `data_model_db` (per tenant): Stores actual business data (WELL, FACILITY, etc.)
- Redis: Optional cache layer for reference data + metadata

#### 8. Hierarchy Tree with Recursive Delete
`deleteHierarchyNodeWithChildrenImpl()` uses recursive traversal to build a list of all descendant `HIERARCHY_DESC_ID` values, deletes all `HIERARCHY_LEVEL` rows for all descendants, then deletes `HIERARCHY_DESC` rows bottom-up via `deleteHierarchyDescRecursively()`. No `ON DELETE CASCADE` in DB — all cascade logic is in Java.

#### 9. XMGMT Cross-Tenant Save (`saveOrUpdateXmgmt`)
Allows a managing tenant (`xmgmtTenantId`) to save data on behalf of an owner tenant (`ownerTenantId`). Validates cross-tenant permission via `TENANT_XREF` + `XMGMT_BUSINESS_OBJECT` join before switching `executionContext.getTenantContext().setTenantId(ownerTenantId)`. After save, records the operation in `XMGMT_DATA` table under the managing tenant's schema.

### 19.11 Gaps / Mismatch

| Issue | Detail |
|---|---|
| No global `ExceptionMapper` | Every method has its own `try/catch (Throwable)` — inconsistent error format risk across endpoints |
| No CDI injection | Services are singleton-factory pattern (`.getInstance()`) — not CDI-managed, harder to unit test/mock |
| JAX-RS annotation duplication | All `@GET`, `@POST`, `@Path`, `@Consumes`, `@Produces` duplicated on both interface and implementation — must be manually kept in sync |
| `validateAndConvertJSONB` double-parses JSON | After `JsonbValidationUtils` parses the JSON value, `validateAndConvertJSONB` re-parses it with Jackson `ObjectMapper` — unnecessary overhead per call |
| `%25` URL-encoded unit names | Unit symbol `%` stored as `%25` in some user settings; `UnitHelper.getUnitNameByMapkey()` returns raw string — `UnitUtility` has no mapping for `%25` → throws `RuntimeException` |
| Missing `@Path` class-level annotation | `MainserviceImpl`, `SearchRestServiceImpl`, `MetadataRestServiceImpl` have no class-level `@Path` — relies on JAX-RS scanning picking up method-level annotations only |
| `compositePrimaryKeyIndex` unused variable | `AbstractDynamicDAOImplForDelete` declares `compositePrimaryKeyIndex = 0` at lines 301 and 338 — value never used |

### 19.12 Simple Explanation

**For saving data (e.g., creating a Well):**

> User calls `POST /save` with JSON → `MainserviceImpl.saveOrUpdate()` validates JWT token via Keycloak → checks timezone and language in payload → routes to `saveOrUpdateImpl()` → `DTOHelper.buildBONameAndBOListMapFromRequestJSON()` reads BO metadata from DB, converts each field to the correct Java type (10 records in parallel), validates FK references via live DB count, validates JSONB mandatory keys → `dynamicWriteService.saveOrUpdate()` builds a dynamic `INSERT INTO well (...) VALUES (?,...)` and executes via JDBC `PreparedStatement.executeBatch()` → updates `BO_SEARCH` index → optionally re-reads saved data → returns `DSPDMResponse` JSON with status + data + messages.

---

**For reading data (e.g., fetching Wells with pagination):**

> User calls `POST /common` with `{"boName":"WELL", "pagination":{"pages":[1], "recordsPerPage":50}}` → `MainserviceImpl.getCustomResponsePost()` validates JWT → builds `BusinessObjectInfo` (BO name, LIMIT 50 OFFSET 0, filters, joins, unit policy) → checks Redis cache for reference data → `AbstractDynamicDAOImplForRead` builds `SELECT w.well_id, w.well_name, ... FROM well w WHERE ... LIMIT 50 OFFSET 0` → executes via JDBC → result rows wrapped in `DynamicDTO` list → `UnitHelper.unitConvertAfterRead()` converts values to user's target units → child BOs recursively fetched if requested → returns `DSPDMResponse` with paginated data + metadata + reference data.

---

**For bulk Excel import:**

> User calls `POST /save/bulk` with multipart form (Excel file + params JSON) → `ImportExportController` parses the file via Apache POI → `ExcelImportJoinUtil.importFromExcel()` preloads all reference data + JSONB metadata into in-memory caches → processes rows, validates each field against metadata, validates JSONB mandatory keys, tracks invalid rows → calls `dynamicWriteService.saveOrUpdate()` in configurable batches → returns response with inserted/updated/invalid record counts.
