- What is Virtual DOM?
  Virtual DOM is a lightweight JavaScript representation of the actual DOM. It's an in-memory 
  representation that React uses to optimize rendering performance.

    How it works:
    When state changes, React creates a new Virtual DOM tree in memory
    React performs diffing - compares new Virtual DOM with previous snapshot
    React identifies only changed elements
    React performs reconciliation - updates ONLY changed parts in the real DOM

    Why it's faster: Virtual DOM minimizes direct DOM manipulation. Instead of updating the entire page, 
    it updates only the modified parts, reducing load times and improving user experience.

    Benefits: Enhanced performance, simplified development (no manual DOM manipulation), and 
    declarative API where you tell React what state you want and it handles the updates."

    Reconciliation is React's hidden engine that makes the declarative programming model possible. 
    It's the process of comparing Virtual DOM trees and updating only what changed in the real DOM, 
    making React fast and efficient
